# 📋 작업 브리프

**원 명령:** # 기술 스택 및 서비스 운영 전략

이 프로젝트는:
모바일 앱이 아닌
“모바일 퍼스트 웹서비스”
로 개발한다.

목표는:
앱 설치 없이
즉시 유입 → 테스트 → 결과 공유 → 결제까지
한 번에 이어지는 구조를 만드는 것이다.

---

# 서비스 플랫폼 전략

플랫폼:

* Responsive Web Service
* Mobile First
* PWA 고려 가능

우선순위:

1. 모바일 웹 UX
2. 빠른 로딩
3. SNS 공유 최적화
4. 인스타그램 유입 최적화

앱 출시:
초기 단계에서는 진행하지 않는다.

이유:

* 인스타 유입은 웹 전환이 더 빠름
* 앱 설치 이탈 방지
* MVP 검증 속도 향상
* 유지보수 비용 절감
* 심리 테스트 특성상 즉시 접근성이 중요함

# 핵심 서비스 전략

우리는:
“심리 검사”
를 만드는 것이 아니다.

우리는:
“사람들이 스스로 설명하지 못했던 감정을

을 만드는 것.

## 요약
전반적인 서비스 기획 및 기술 스택 정의에 대한 브리핑입니다. 핵심은 '심리 테스트'가 아닌, 사용자가 스스로 설명하지 못한 감정을 발견하고 공유하는 '감성 웹 경험' 구축입니다.
이를 위해 모바일 퍼스트 접근 방식과 인스타그램 바이럴 구조를 최우선으로 고려해야 합니다.
