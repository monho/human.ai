## 💡 MOONCO 심층 테스트 MVP UI/UX 디자인 브리프 (모바일 기준)

### **📌 Global Design Guideline**
*   **Primary Color:** `#3B5998` (신뢰감 있는 네이비 계열 - 전문성 강조)
*   **Secondary Color:** `#FFC107` (주의/강조 색상 - 중요 CTA 또는 전환 유도 시 사용)
*   **Background Color:** `#F9F9F9` (깨끗하고 안정적인 느낌 부여)
*   **Typography:** Pretendard (산세리프 계열로 가독성 최우선)
    *   Header: Bold, 24pt 이상
    *   Body Text: Regular, 16pt 기준

### **Screen 1: 스플래시 화면 (Splash Screen)**

| 항목 | 상세 스펙 | 비고/사용자 행동 유도 |
| :--- | :--- | :--- |
| **목적** | 브랜드 노출 및 로딩 시간 커버. 서비스에 대한 기대감 조성. | 2~3초 체류 후 자동 다음 화면 이동. |
| **UI 구성요소** | 1. 중앙 로고 (MOONCO 심볼) 2. 태그라인: "당신의 마음을 비추는 가장 깊은 질문." 3. 로딩 스피너 (심플한 애니메이션) | **로고:** 크기 가이드 준수, 배경에 은은하게 워터마크 처리. **스피너:** 부드러운 원형 패턴 권장. |
| **컬러 적용** | Background: `#FFFFFF` / 로고/텍스트: `#3B5998` | 대비를 주어 시선 집중 유도. |
| **와이어프레임** | 전체 중앙 정렬, 요소 간 충분한 여백 확보 (Whitespace). | 스피너는 중앙 상단에 배치하여 안정감 유지. |

### **Screen 2: 문양 선택 화면 (Onboarding/Identity Selection)**

*   *(참고: '문양'은 심리 테스트의 초기 페르소나 설정 또는 테마 선택으로 해석)*

| 항목 | 상세 스펙 | 비고/사용자 행동 유도 |
| :--- | :--- | :--- |
| **목적** | 사용자에게 주도권을 부여하며, 서비스 경험을 개인화할 여지 제공. (흥미 유발) | 사용자가 3가지 중 하나를 선택하고 '시작하기' 클릭. |
| **UI 구성요소** | 1. 제목: "어떤 이야기로 시작하시겠어요?" 2. 옵션 카드 3개 (예: 별자리, 동물, 색상 등 테마) 3. 각 카드는 이미지 + 명칭 + 짧은 설명 포함. 4. CTA 버튼: "선택하고 테스트 시작하기" | **카드 디자인:** 모서리 라운딩(Radius: 12px), 선택 시 `#CCE5FF` 배경색 변경 및 테두리 강조. |
| **컬러 적용** | Background: `#F9F9F9` / 옵션 카드 배경: `#FFFFFF` / 활성화 상태 CTA: `#3B5998` (Primary) | 전체적으로 부드럽고 선택하기 쉬운 톤 유지. |
| **와이어프레임** | 상단 제목 $\rightarrow$ 중앙 3열 그리드 배치(카드) $\rightarrow$ 하단 고정 CTA 버튼. | 시선이 자연스럽게 옵션 카드들로 분산되도록 설계. |

### **Screen 3: 질문 진행 화면 (Questionnaire Flow)**

*   *(핵심 로직 구현 단계)*

| 항목 | 상세 스펙 | 비고/사용자 행동 유도 |
| :--- | :--- | :--- |
| **목적** | 사용자에게 부담 없이, 높은 몰입도를 유지하며 답변을 수집. (가장 중요) | 사용자가 답변을 선택하고 '다음 질문' 클릭할 때까지 체류. |
| **UI 구성요소** | 1. 진행률 표시줄 (Progress Bar): 현재 질문 X/총 Y개. 2. 문항(질문): 큰 타이포로 명확하게 제시. 3. 답변 옵션: 시각적 선택지 제공 (예: 이모티콘 기반, Likert Scale). 4. 네비게이션 버튼: "이전", **"다음 질문"** | **답변 구조:** 5점 척도(매우 그렇지 않다 $\leftrightarrow$ 매우 그렇다)의 라디오 버튼 그룹 또는 감정 아이콘 선택 방식 추천. |
| **컬러 적용** | Background: `#FFFFFF` / 활성화된 답변 옵션 배경: 연한 녹색 계열 (`#E8F5E9`)으로 긍정적 피드백 제공. CTA: `#3B5998`. | 질문은 간결하게, 선택지는 직관적으로 배치하여 인지 부하 최소화. |
| **와이어프레임** | 상단 (진행률) $\rightarrow$ 중앙 (질문 텍스트 영역) $\rightarrow$ 중하단 (답변 옵션 그리드/선택 필드) $\rightarrow$ 하단 고정 CTA 버튼. | 답변 선택 시, 해당 선택지가 명확히 '체크'되어야 함. |

### **Screen 4: 감정 변화 기록 화면 (Emotional Check-in)**

*   *(심화 분석 및 데이터 수집 단계 - 인터랙티브 요소 강조)*

| 항목 | 상세 스펙 | 비고/사용자 행동 유도 |
| :--- | :--- | :--- |
| **목적** | 사용자가 자신의 현재 감정 상태를 메타적으로 인지하고 기록하게 함. (재방문율 및 데이터 축적 목표) | 3~5가지 핵심 감정 아이콘 중 가장 가까운 것을 '클릭'하여 선택 후, 간단한 메모 입력 유도. |
| **UI 구성요소** | 1. 제목: "지금, 나를 표현한다면?" 2. 감정 그리드: 대표적 감정 (슬픔, 불안, 평온 등)을 상징하는 대형 아이콘 5~7개 배치. 3. 선택된 아이콘 하이라이트 및 애니메이션 피드백. 4. 보조 입력창: "느낀 점(선택 사항)" - 텍스트 필드 (최대 100자). 5. CTA 버튼: "결과 확인하기" | **감정 아이콘:** 단순한 일러스트레이션 사용, 감정의 스펙트럼을 시각화하는 것이 중요함. |
| **컬러 적용** | Background: `#F9F9F9` / 선택된 아이콘 주변으로 은은하게 빛나는 '후광 효과' (Glow Effect) 부여. CTA: Primary Color와 대비되는 차분한 색상 사용. | 감정 표현에 대한 심리적 장벽을 낮추기 위해, 강렬함보다는 부드러움을 유지해야 함. |
| **와이어프레임** | 상단 제목 $\rightarrow$ 중앙 대형 아이콘 그리드 (선택 영역) $\rightarrow$ 하단 메모 입력 필드 및 CTA 버튼. | 사용자가 감정 선택에만 집중하도록 여백을 극대화합니다. |

### **Screen 5: 무료 결과 화면 (Free Result Summary)**

*   *(미끼 상품 - 가치 제공으로 신뢰 구축)*

| 항목 | 상세 스펙 | 비고/사용자 행동 유도 |
| :--- | :--- | :--- |
| **목적** | 테스트에 대한 기본적인 인사이트를 무료로 제공하여 만족감을 주고, 다음 단계(유료)로의 자연스러운 연결 고리 만들기. | 결과 요약 확인 후, '더 깊이 알고 싶다면' 버튼을 통해 Screen 6으로 이동 유도. |
| **UI 구성요소** | 1. 최종 점수/키워드: "당신은 [OOO] 유형입니다." (가장 크게 표시) 2. 핵심 요약 3가지 (Benefit Point): 전문 용어 대신 쉬운 문구로 설명. 3. 가이드 문구: "더 깊이 있는 원인과 해결책을 알고 싶으신가요?" 4. CTA 버튼 1: **"무료 리포트 다운로드 (제한적)"** / CTA 버튼 2: **"심층 분석 리포트 보기 (유료)"** | 두 개의 대비되는 CTA를 배치하여 의사결정 구조 만들기. |
| **컬러 적용** | Background: `#FFFFFF` / 핵심 결과 키워드 배경 강조: 밝고 희망적인 톤의 파스텔 계열 사용 (`#B3E5FC`). 유료 CTA 버튼은 Primary Color를 사용하여 시선 집중. | 무료 내용은 충분히 만족스럽게 느껴지도록 디자인하되, 반드시 다음 행동을 유도해야 함. |
| **와이어프레임** | 상단 (결과 제목) $\rightarrow$ 중앙 (핵심 3가지 요약 박스 - 카드 형태) $\rightarrow$ 하단 두 개의 CTA 버튼이 분리 배치됨. |

### **Screen 6: 유료 전환 화면 (Paywall/Upgrade)**

*   *(수익화 핵심 지점)*

| 항목 | 상세 스펙 | 비고/사용자 행동 유도 |
| :--- | :--- | :--- |
| **목적** | 무료 결과의 한계점을 명확히 제시하고, 유료 리포트가 제공할 압도적인 가치를 설득. | 사용자가 '구매하기' 버튼을 누르게 함 (결제창으로 이동). |
| **UI 구성요소** | 1. 강력한 헤드라인: "진짜 나를 만나려면, 심층 분석이 필요합니다." 2. 비교 테이블/박스: [무료] vs [유료]. 유료 박스에 'X' 표시된 기능(예: 원인 패턴 추적, 맞춤 행동 지침)을 강조. 3. 가격 정보 명시 (Subscription Model 제시). 4. CTA 버튼: **"OOO 심층 리포트 구매하기 (₩XX,XXX)"** | 비교를 통해 가치 차이를 극대화하고, 결제 장벽을 낮추기 위해 '일회성' 옵션도 함께 제시 고려. |
| **컬러 적용** | Background: `#FFF3E0` (경고/주의를 나타내는 따뜻한 톤으로 긴장감 조성). 유료 CTA 버튼은 가장 강렬한 대비색 사용. | 전반적으로 '정보 제공'보다는 '설득'의 느낌이 강해야 합니다. |
| **와이어프레임** | 상단 (문제 제기) $\rightarrow$ 중앙 비교 구조 (대비 효과 극대화) $\rightarrow$ 하단 최종 결제 CTA 버튼 고정. |

### **Screen 7: 유료 심층 리포트 화면 (Premium Report View)**

*   *(궁극의 가치 전달 - 사용자가 '돈을 낸 이유'를 충족)*

| 항목 | 상세 스펙 | 비고/사용자 행동 유도 |
| :--- | :--- | :--- |
| **목적** | 높은 정보 밀도를 유지하며, 전문성과 신뢰감을 극대화하여 재방문 및 서비스 지속 이용을 유도. | 리포트 열람 후 '다음 단계 가이드'를 통해 추가 서비스(예: 코칭 연결)로 안내. |
| **UI 구성요소** | 1. 종합 분석 결과 제목 (전문적 명칭 사용). 2. 파트별 분할 구조: (A) 근본 원인 분석, (B) 심리 패턴 매핑, (C) 맞춤 행동 가이드라인 (구체적인 'Action Plan' 제시). 3. 차트/그래프 시각화 요소 다수 포함. 4. 최종 CTA 버튼: **"나의 변화를 위해 [전문가 연결]하기"** | 단순 텍스트 나열 금지. 반드시 인포그래픽, 도식화된 그래프 활용 필수. |
| **컬러 적용** | Background: `#FFFFFF` / 전문 데이터를 나타내는 차트/라인은 Primary Color 계열을 메인으로 사용하고, 핵심 개선점(Action)에는 Secondary Color를 사용하여 '해결책'임을 명확히 함. | 학술적이고 정돈된 느낌 (Medical/Academic UI). |
| **와이어프레임** | 상단 헤더 (분석 주제) $\rightarrow$ 섹션 1: 그래프 및 원인 분석 (시각 자료 중심) $\rightarrow$ 섹션 2: 행동 가이드라인 목록화 (체크리스트 형태) $\rightarrow$ 하단 최종 CTA 버튼. |
